# io.github.aguilarguisado.siestadeck.sdPlugin/ — the Stream Deck plugin bundle

This directory **is** the plugin from Stream Deck's perspective. Everything in here is what gets symlinked by `npm run link` and packed by `npm run pack`. The TypeScript source in `../src/` compiles into `bin/plugin.js`.

## Layout

- `manifest.json` — plugin metadata + action declarations. Stream Deck reads this on plugin load.
- `bin/plugin.js` — Rollup output, gitignored. Don't edit by hand. `manifest.json:10` points `CodePath` here.
- `bin/resvg.wasm` — copied in by `rollup.config.mjs` so `rasterize.ts` can `readFileSync` it next to the bundle.
- `imgs/` — generated by `npm run icons` from `../assets/icons/*.svg`. Don't edit by hand; edit the SVGs.
- `pi/*.html` — Property Inspector UI for each action. Plain HTML using Elgato `sdpi-components` web components.

## Manifest ↔ action class binding

Each action in `manifest.json` has a `UUID` that **must** match the `@action({ UUID: ... })` decorator on its TypeScript class. Current mapping:

| Manifest UUID | Class | File |
|---|---|---|
| `io.github.aguilarguisado.siestadeck.quota-meter` | `QuotaMeter` | `src/actions/quotaMeter.ts` |
| `io.github.aguilarguisado.siestadeck.extra-usage` | `ExtraUsage` | `src/actions/extraUsage.ts` |
| `io.github.aguilarguisado.siestadeck.active-model` | `ActiveModel` | `src/actions/activeModel.ts` |
| `io.github.aguilarguisado.siestadeck.switch-account` | `SwitchAccount` | `src/actions/switchAccount.ts` |
| `io.github.aguilarguisado.siestadeck.login-logout` | `LoginLogout` | `src/actions/loginLogout.ts` |
| `io.github.aguilarguisado.siestadeck.attention` | `Attention` | `src/actions/attention.ts` |

Adding an action requires **all three**: a manifest entry, a class with the matching `@action` UUID, and a `streamDeck.actions.registerAction(...)` call in `src/plugin.ts:17-21`.

## Property Inspector HTML

Each action's `PropertyInspectorPath` (e.g. `pi/quotaMeter.html`) loads when the user selects the key in the Stream Deck app. Conventions:

- Use Elgato's `sdpi-components` (`<sdpi-item>`, `<sdpi-select>`, `<sdpi-checkbox>`, etc.).
- Form field `setting` attributes map to the action's `TSettings` type. Numeric inputs come back as **strings** — coerce with `Number()` in the action.
- For account dropdowns, set `datasource="getAccounts"` or `datasource="getAccountsIncludingActive"`. Stream Deck dispatches a `sendToPlugin` with `payload.event = "<datasource>"`; the action's `onSendToPlugin` should call `handleAccountDatasource(ev)` from `src/services/piDatasources.ts:14-35`. The handler replies via `streamDeck.ui.sendToPropertyInspector({ event, items })`.

## Manifest constraints

- `OS: [{ Platform: "mac", MinimumVersion: "12" }]` — macOS 12+ only. Helvetica path in `src/render/rasterize.ts:8` assumes this.
- `Nodejs: { Version: "20", Debug: "enabled" }` — Stream Deck spawns Node 20 and exposes the inspector. Connect via `chrome://inspect` or the Stream Deck dev tools URL printed in logs.
- `SDKVersion: 2`, `Software.MinimumVersion: "6.5"` — bumping these breaks older Stream Deck app installs.
- Bumping plugin `Version` is required for users to receive updates from a `.streamDeckPlugin` install.

## Build outputs

`bin/` and `imgs/` are both gitignored (see project `.gitignore`). PNGs are regenerated from `assets/icons/*.svg` by `npm run icons`, which runs as a `prebuild` hook. Always run `npm run icons` after editing an SVG, and `npm run validate` after editing `manifest.json`. CI rebuilds them on every push.
